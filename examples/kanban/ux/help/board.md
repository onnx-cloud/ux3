# Kanban Board - Organize Your Work

The Kanban board visualizes your workflow with lanes (columns) and tasks (cards). Drag tasks between lanes to track progress.

## Board Layout

### Lanes (Columns)
- Each vertical column represents a stage (e.g., To Do, In Progress, Done)
- Lanes show task counts and can be collapsed to save space
- Customize lanes via **Lane Manager**

### Tasks (Cards)
- Each card represents a unit of work
- Shows task title, assignee, priority, and labels
- Click to view full details or edit

## Core Actions

### Managing Tasks

| Action | How |
|--------|-----|
| **Create Task** | Click "+ Task" button in a lane |
| **Edit Task** | Click a task card to open details |
| **Move Task** | Drag card between lanes |
| **Assign Task** | Click on a task → set assignee |
| **Add Labels** | Click on a task → add label tags |
| **Set Priority** | Click on a task → choose priority level |
| **Complete Task** | Drag task to "Done" lane |
| **Delete Task** | Open task → click delete button |

### Filtering & Searching

- **Filter by Assignee**: See tasks assigned to specific people
- **Filter by Priority**: Focus on high-urgency work
- **Filter by Labels**: Group tasks by category
- **Hide Archived**: Toggle visibility of completed tasks
- **Search**: Use the search box to find tasks by title

### Lane Management

- **Collapse Lane**: Click the arrow to minimize a column
- **Reorder Lanes**: Drag lane header to change position
- **Add Lane**: Click "+ Lane" to create a new stage
- **Configure Lanes**: Click **Lane Manager** button for advanced settings

## Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| `N` | Create new task |

| `/` | Global search |
| `?` | Show help |

## Tips for Success

💡 **Keep lanes focused**: Fewer, well-defined lanes work better than many

💡 **Limit WIP**: Set maximum tasks per lane (e.g., max 5 in progress)

💡 **Use labels**: Tag tasks by category, team, or component

💡 **Archive regularly**: Move completed tasks to archive to reduce clutter

💡 **Color code**: Use colors for different task types or priorities

## Common Workflows

### Agile/Scrum
- **Lanes**: Backlog → Sprint → In Progress → In Review → Done
- **Cards**: User stories with story points
- **Cycle**: 1-2 week sprints

### Kanban Flow
- **Lanes**: To Do → In Progress → Waiting → Done
- **Cards**: Issues and features
- **Flow**: Continuous, pull-based

### Issue Triage
- **Lanes**: Reported → Verified → In Progress → Deployed
- **Cards**: Bug reports with severity

## Getting More Done

1. **Define your workflow** with appropriate lanes
2. **Set WIP limits** to prevent overload
3. **Review daily** to identify blockers
4. **Archive completed** tasks frequently
5. **Use labels** to categorize and find work
