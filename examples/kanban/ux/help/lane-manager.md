# Lane Manager - Customize Your Workflow

Lanes are the columns in your Kanban board. Use this view to create, edit, and organize lanes to match your workflow.

## What Are Lanes?

Lanes represent stages in your workflow. Common examples:
- **To Do** → **In Progress** → **In Review** → **Done**
- **Backlog** → **Ready** → **Doing** → **Testing** → **Released**
- **New** → **Assigned** → **In Progress** → **Blocked** → **Closed**

## Managing Lanes

### Create a Lane
1. Click **"+ Add Lane"** button
2. Enter lane name (e.g., "In Review")
3. Choose optional color and icon
4. Click **Create**

### Edit a Lane
1. Click the lane name in the list
2. Update name, color, or settings
3. Click **Save**
4. Changes appear on board immediately

### Reorder Lanes
- **Drag** lanes to change position
- Order determines left-to-right appearance on board
- Move completed stages to the right

### Delete a Lane
1. Click the **Delete** button on the lane
2. Choose what to do with tasks in that lane:
   - Move to different lane
   - Archive them
   - Delete them
3. Confirm deletion

## Lane Settings

### Basic Properties
- **Name**: Display name on board
- **Description**: Internal notes about the lane
- **Color**: Visual identification
- **Icon**: Optional visual indicator

### Workflow Settings
- **WIP Limit**: Max tasks allowed (e.g., max 5 "In Progress")
- **Auto-Archive**: Archive tasks after days idle
- **Block Progress**: Prevent moving to this lane if conditions aren't met
- **Notification**: Alert team when tasks enter lane

### Display Options
- **Collapsed**: Show/hide lane on board by default
- **Hidden**: Don't show on board (archival stage)
- **Expand All**: Show all tasks without scrolling
- **Task Count**: Display number of tasks

## Best Practices

### Workflow Design
💡 **Keep it simple**: 3-5 lanes often work better than 10+

💡 **Match your process**: Align lanes with actual stages in your work

💡 **Use WIP limits**: Prevents bottlenecks and encourages flow

💡 **Iterate**: Your lanes will evolve as you use them

### Naming Lanes
Good names are:
- ✅ "To Do"
- ✅ "In Progress"
- ✅ "Waiting for Review"
- ❌ "Stage 1"
- ❌ "Pending"
- ❌ "X"

### Color Coding
- Use colors to highlight important stages
- Red for blocking lanes, Green for completion
- Keep it consistent across projects

## Example Workflows

### Software Development
1. **Backlog** - Future work
2. **To Do** - This iteration
3. **In Progress** - Currently working
4. **In Review** - Waiting for approval
5. **Done** - Completed

### Bug Triage
1. **Reported** - New issues
2. **Verified** - Confirmed bugs
3. **Assigned** - Given to developer
4. **Fixed** - Code changes made
5. **Deployed** - Released to production

### Content Creation
1. **Ideas** - Brainstorming
2. **Writing** - Draft in progress
3. **Review** - Editor feedback
4. **Published** - Live content
5. **Archived** - Old posts

## Advanced Features

### Templates
- Save your lane configuration as a template
- Reuse lane setup across multiple boards
- Speeds up new project creation

### Lane Dependencies
- Create blocking relationships
- Example: Can't move to "Deploy" until "Review" is approved
- Prevents skipping important steps

### Automation
- Auto-move tasks based on age or status
- Auto-archive completed tasks
- Notify team when lanes fill up

---

**Tip**: Review and refine your lanes every sprint. A good workflow evolves with your needs!
