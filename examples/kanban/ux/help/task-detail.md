# Task Detail - View & Edit a Task

This view shows all information about a task and allows you to edit it. Make changes on the fly to keep your work organized.

## Task Information

### Basic Details
- **Title**: The task name or summary
- **Description**: Full details about what needs to be done
- **Status**: Which lane the task is in (To Do, In Progress, Done, etc.)
- **ID**: Unique task identifier

### Task Properties

| Property | Purpose |
|----------|---------|
| **Assignee** | Who is responsible for this task |
| **Priority** | Urgency level (Low, Normal, High, Urgent) |
| **Due Date** | When the task should be completed |
| **Labels** | Tags for categorization and filtering |
| **Time Estimate** | Expected effort (in hours or story points) |
| **Time Spent** | Actual hours worked |

## Editing Tasks

### How to Edit

1. **Click any field** to edit it inline, or
2. **Click "Edit"** button to open full editor
3. **Make changes** to any property
4. **Save** automatically or click Save button
5. **Changes sync** to the board instantly

### Quick Edits

- **Drag to change status**: Move between lanes
- **Click to reassign**: Select a new team member
- **Type to update**: Edit title or description
- **Press Enter**: Save changes

## Task Actions

| Action | Use Case |
|--------|----------|
| **Add Comment** | Discuss the task with team members |
| **Attach File** | Link documents or images |
| **Add Subtask** | Break down into smaller Steps |
| **Create Link** | Reference related tasks or tickets |
| **Change History** | View all edits and updates |

## Collaboration Features

### Comments
- Add notes about progress or blockers
- @ mention teammates to notify them
- Comment history shows who said what and when

### Activity Log
- See all changes made to the task
- Who changed what and when
- Useful for tracking modifications

### Assignment
- Assign to one person (or multiple people if supported)
- Assignee gets notified when assigned
- Shows avatars of all assigned users

## Priority Levels

| Level | Color | When to Use |
|-------|-------|-------------|
| **Urgent** | Red | Must do today |
| **High** | Orange | This week |
| **Normal** | Blue | Standard work |
| **Low** | Green | Nice to have |

## Tips

💡 **Clear titles**: Write titles that explain the work, not "Fix it"

💡 **Good descriptions**: Include context, acceptance criteria, and links

💡 **Due dates**: Set realistic dates based on complexity

💡 **Regular updates**: Keep task status in sync with actual progress

💡 **Use labels**: Consistent labeling helps with searching and filtering

## Common Task Types

### Bug Fix
- Title: Describe the problem
- Priority: Depends on severity
- Labels: "bug", "priority:high"

### Feature
- Title: "Implement [feature name]"
- Description: User story or feature spec
- Labels: "feature", "epic:[name]"

### Documentation
- Title: "Document [feature]"
- Priority: Usually normal
- Labels: "doc", "documentation"
