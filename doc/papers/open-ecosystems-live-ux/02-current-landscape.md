This chapter surveys the current landscape of UI metadata, plugin systems, and live development tools.

There is a proliferation of metadata formats, but few focus on executable UI artifacts. Existing plugin systems often expose components and services, but they do not always publish machine-readable contracts for composition.

UX3’s plugin model is notable because it supports MCP tools, runtime resources, and compile-time schema extensions. This makes it a strong candidate for an open ecosystem that values discoverability and interoperability.

The chapter also identifies gaps. Many ecosystems lack a shared schema for stateful UI artifacts, and many live compilation systems treat metadata as secondary to runtime code. UX3 can address these gaps by elevating declarative metadata and plugin-hosted tools.
